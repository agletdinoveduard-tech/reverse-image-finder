# Ошибки и решения

## `/api/health` показывает `hasSearchApi: false`

Worker не видит ключ SearchAPI.

Проверь:

```text
Worker → Settings → Variables and Secrets
```

Должно быть строго:

```text
SEARCHAPI_KEY
```

Тип лучше выбрать **Secret**.

## `/api/health` показывает `hasTempKv: false`

Не подключён KV binding.

Проверь:

```text
Worker → Settings → Bindings
```

Должно быть:

```text
Variable name: TEMP_IMAGES
```

## Кнопка загрузки не работает

В текущей версии используется обычное видимое поле:

```text
<input type="file">
```

Проверяй программу не в Cloudflare Preview, а по прямой ссылке:

```text
https://твой-worker.workers.dev/
```

После обновления кода нажми:

```text
Ctrl + F5
```

## Временная ссылка на изображение отдаёт 404

Проверь, что KV binding называется именно:

```text
TEMP_IMAGES
```

Также учитывай, что Workers KV может иметь задержку распространения. В коде добавлена пауза и повторные попытки SearchAPI.

## SearchAPI ничего не нашёл

Это не всегда ошибка. Возможные причины:

- изображение нигде не опубликовано;
- изображение сильно изменено, обрезано или улучшено;
- точных совпадений нет в индексе;
- выбран слишком узкий режим поиска.

Попробуй режим:

```text
Максимум
```

## Страница растянута за экран

В текущей версии CSS ограничивает горизонтальное растягивание:

```css
html, body { overflow-x: hidden; }
```

Если какой-то блок всё равно вылезает, обычно причина — очень длинный URL в карточке результата. В таком случае пришли скрин проблемного места.

## GitHub Pages не запускает поиск

Так и должно быть. GitHub Pages — статический хостинг. Рабочий поиск должен запускаться через Cloudflare Worker.
